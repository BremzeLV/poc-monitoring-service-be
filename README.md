# Running the POC
- Clone repo
- Run `cp env-example-relational .env` in root
- Run `docker-compose up --build`
- Visit http://localhost:3000/
- Enjoy or suffer - depends on you!

### Additional info about the project:
- Tests are not written but would like to see at least some E2E tests for the functionality;
- *BUT* the service is testable via 3 mock devices that are booted up once docker is ran. All are callable by http or gRPC. POC service will try to call them from seed data. Some of them will fail and some of them will be successful. The devices responses are randomized to give some flakiness, super simple implementations, did not go deep into them; You can add `device-mock-2:3002` with `http` or `gRPC` protocol and see how it goes.

| Device | HTTP  | gRPC | Example with port | 
| ------------- | ------------- | ------------- | ------------- |
| device-mock-1 | 3001 | 50051 | device-mock-1:3001 |
| device-mock-2 | 3002 | 50052  | device-mock-2:50052 |
| device-mock-3 | 3003  | 50053  | device-mock-3:3003 |

- AI was used for UI markdown and styling generation;
- Used Kafka so its simple to scale. NestJS does not support batch processing via Kafka natively therefore using one-by-one message consumption but consumer could be built so it would consume in batches.

NestJS REST API boilerplate for a typical project

## Features

- [x] Database. Support [TypeORM](https://www.npmjs.com/package/typeorm) and [Mongoose](https://www.npmjs.com/package/mongoose).
- [x] Seeding.
- [x] Config Service ([@nestjs/config](https://www.npmjs.com/package/@nestjs/config)).
- [x] Mailing ([nodemailer](https://www.npmjs.com/package/nodemailer)).
- [x] Sign in and sign up via email.
- [x] Social sign in (Apple, Facebook, Google).
- [x] Admin and User roles.
- [x] Internationalization/Translations (I18N) ([nestjs-i18n](https://www.npmjs.com/package/nestjs-i18n)).
- [x] File uploads. Support local and Amazon S3 drivers.
- [x] Swagger.
- [x] E2E and units tests.
- [x] Docker.
- [x] CI (Github Actions).
